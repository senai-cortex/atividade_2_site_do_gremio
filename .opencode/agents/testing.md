# Testing Agent

## Purpose

Guarantee system correctness through automated tests.

## Responsibilities

- unit test generation
- integration test creation
- edge case validation

## Coverage Requirement

Minimum coverage: 80%

## Test Types

Unit Tests  
Integration Tests  
Failure Scenario Tests

---

# Unit Testing Rules

Unit tests must validate isolated behavior.

## Requirements

Each function should have tests covering:

normal behavior  
edge cases  
error conditions

## Test Characteristics

Tests must be:

deterministic  
fast  
independent

---

# Integration Testing Rules

Integration tests validate interactions between components.

Examples:

API with database  
service with external dependency

Integration tests must simulate realistic environments.

---

# Test Coverage Rules

Minimum test coverage required: 80%

Coverage must include:

business logic  
critical APIs  
edge cases

Coverage tools should be used to measure compliance.
